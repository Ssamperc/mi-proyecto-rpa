Quiero que desarrolles una aplicación web completa y funcional, totalmente ejecutable en Replit, que permita ejecutar un sistema RPA desde una interfaz web. A continuación te doy todos los requisitos detallados. Necesito que generes todo el código listo para usar: backend, frontend, rutas, procesadores, templates, manejo de archivos y recomendaciones finales.

====================================================================
REQUISITOS GENERALES DEL PROYECTO – SISTEMA RPA WEB

La aplicación debe ser una web app con estas características:

Framework backend: Flask (Python)

Frontend: HTML + CSS + JS sencillo (Bootstrap opcional)

Ejecución: 100% funcional en Replit sin configuraciones externas

Interacción principal:

Un formulario donde el usuario pueda subir un archivo Excel/CSV.

El sistema debe guardar ese archivo en una carpeta data/.

Luego debe ejecutar EL MOTOR RPA (que tú vas a programar).

Debe mostrar en pantalla un panel de resultados.

Debe permitir descargar los archivos generados en output/.

====================================================================
FUNCIONALIDADES QUE DEBE INCLUIR EL RPA

El sistema RPA debe realizar lo siguiente:

1. LECTURA Y VALIDACIÓN DE PEDIDOS

El archivo subido por el usuario contendrá columnas como:

ID_Pedido

Fecha_Pedido

Nombre_Cliente

Correo_Cliente

Direccion_Envio

SKU

Nombre_Producto

Cantidad

Precio_Unitario

Valor_Total

Debe validar:

Campos obligatorios vacíos

Cantidad y Valor_Total deben ser numéricos

Correo electrónico debe tener un formato válido

Valor_Total > 0

Debe generar automáticamente:

output/PedidosValidados.xlsx

output/ErroresRPA.xlsx

Y retornar estadísticas:

Total pedidos

Válidos

Inválidos

2. SIMULACIÓN DE REGISTRO EN SAG

No existe conexión real a SAG, así que debes simularlo.

El RPA debe:

Recibir los pedidos validados

“Registrar” cada uno y generar:

Estado_Registro

Fecha_Registro

Debe generar:

output/PedidosRegistradosSAG.xlsx

Con estadísticas:

Registrados exitosamente

Fallidos (puedes simular uno aleatorio)

3. NOTIFICACIONES POR CORREO (SIMULACIÓN)

Debe simular envíos de correo:

Mostrar en consola o guardar en archivo:

Destinatario

Asunto

ID_Pedido

Debe generar:

output/LogCorreos.txt

Y estadísticas:

Correos enviados

Correos fallidos

4. DASHBOARD Y REPORTES

Debe generarse:

output/ReporteProcesados.xlsx

output/DashboardRPA.xlsx

El dashboard debe contener:

Total de pedidos recibidos

Total válidos

Total inválidos

Tasa de validación (%)

Tasa de registro en SAG (%)

Tiempos aproximados

Gráficos simples (pie/barras)
